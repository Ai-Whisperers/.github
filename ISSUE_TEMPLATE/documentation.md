## Documentation Standards Reference

Our documentation should:
1. **Be accurate** - Match actual behavior
2. **Be complete** - Cover all important aspects
3. **Be clear** - Use simple, direct language
4. **Be consistent** - Follow established patterns
5. **Include examples** - Show, don't just tell
6. **Be accessible** - Use proper formatting and structure
7. **Be maintainable** - Easy to update

### Formatting Guidelines
- Use Markdown with consistent heading levels
- Include code blocks with language specification
- Use tables for comparisons
- Add screenshots for UI elements
- Include warning/note boxes for important information

### Language & Tone
- Use active voice
- Be concise but thorough
- Avoid jargon when possible
- Define technical terms on first use
- Use inclusive language
